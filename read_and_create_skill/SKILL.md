# read_and_create_skill

## 说明

本技能用于基于用户提供的 CSV 表格自动创建 Agent 技能包。用户将填好的表格放在 `data/` 文件夹中，提供表格文件名后，本技能会读取表格内容并生成完整的 skill 技能包。

## 使用流程

### 1. 表格文件准备

用户需将填好的表格文件（.csv 格式）放置在 `read_and_create_skill/data/` 目录下。

### 2. 用户输入格式

用户会以如下形式提供表格名称：
- "我有个叫 xxx 的表格"
- "帮我基于 xxx 表格生成技能"
- "用 xxx.csv 创建一个新技能"

**你需要从用户输入中提取表格文件名**（不带 .csv 后缀）。

### 3. 读取表格内容

使用 `read_and_create_skill/scripts/csv_to_prompt.py` 脚本读取表格内容：

```bash
python3 read_and_create_skill/scripts/csv_to_prompt.py <表格文件名>
```

**注意**：
- 只需提供文件名，不需要 `.csv` 后缀
- 脚本会自动在 `data/` 目录下查找对应文件
- 脚本会将表格的字段名和内容拼接为结构化提示词

### 4. 解析表格字段

CSV 表格包含以下字段：

| 字段名 | 说明 |
|--------|------|
| SKILL名称 | 生成的 skill 文件夹名称（建议英文，小写，用连字符分隔） |
| 使用场景描述 | 何时触发此技能，包含触发关键词和条件 |
| SOP流程 | 业务执行的标准操作流程步骤 |
| 步骤分流逻辑 | 不同场景下的分支判断规则 |
| 需用户反馈内容 | 执行任务所需的必填信息列表 |
| 反馈内容缺失是否反问 | true/false，缺失时是否主动反问用户 |
| 接口工具（开发提供） | API 接口的调用方式（curl 示例） |
| 接口入参（开发提供） | API 请求的参数说明（JSON 格式） |
| 接口出参（开发提供） | API 返回的数据格式说明 |

### 5. 创建技能包

根据表格内容，在指定目录（默认为 `workspace/skills/`）创建技能包，结构如下：

```
skills/
└── {SKILL名称}/
    ├── SKILL.md              # 技能主文件
    └── references/
        └── api_reference.md  # API 接口参考文档
```

#### 5.1 生成 references/api_reference.md

将表格中的以下内容写入此文件：
- 接口工具（开发提供）
- 接口入参（开发提供）
- 接口出参（开发提供）

**此文件是接口调用的权威参考，当工具调用出错时，必须优先读取此文件排查问题。**

#### 5.2 生成 SKILL.md

基于表格内容生成技能主文件，结构如下：

```markdown
# {SKILL名称}

## 说明

{使用场景描述}

## 使用时机

{从使用场景描述中提取触发条件和关键词}

## 执行步骤

{SOP流程内容}

## 分流逻辑

{步骤分流逻辑内容}

## 需收集的信息

{需用户反馈内容}

{如果反馈内容缺失是否反问为 true，添加：缺失时主动反问用户以获取信息}

## 工具调用

- 接口文档位置：`references/api_reference.md`
- **当工具调用出错时，优先读取 `references/api_reference.md` 排查问题**

## 响应示例

{根据接口出参格式，提供自然的响应话术示例}
```

## 完整执行示例

假设用户说："我有个叫 policy_search 的表格，帮我生成技能"

执行步骤：

1. **提取文件名**：`policy_search`

2. **读取表格**：
   ```bash
   python3 read_and_create_skill/scripts/csv_to_prompt.py policy_search
   ```

3. **解析输出**：脚本会返回结构化的表格内容，包含所有字段信息

4. **创建目录结构**：
   ```
   skills/
   └── policy-search/
       ├── SKILL.md
       └── references/
           └── api_reference.md
   ```

5. **生成 api_reference.md**：写入接口工具、入参、出参

6. **生成 SKILL.md**：基于表格其他字段生成完整技能文档

## 错误处理

执行过程中可能遇到以下情况：

1. **表格文件不存在**：提示用户检查 `read_and_create_skill/data/` 目录下是否有该文件
2. **CSV 格式错误**：确保文件使用 UTF-8 编码，字段间用逗号分隔
3. **字段缺失**：如果表格缺少必要字段（如 SKILL名称、使用场景描述），需提示用户补充
4. **脚本执行失败**：检查 Python 环境是否正常，脚本路径是否正确

## 注意事项

1. **文件名处理**：从用户输入提取的文件名可能包含或不含 `.csv` 后缀，需统一处理
2. **目录命名**：SKILL名称建议转为小写，空格替换为连字符（如 `Policy Search` → `policy-search`）
3. **API 参考**：`references/api_reference.md` 必须包含完整的接口信息，作为排错第一参考
4. **重复生成**：如果目标技能目录已存在，询问用户是覆盖还是增量更新

## 输出要求

生成的技能包必须满足：

1. **结构完整**：包含 `SKILL.md` 和 `references/api_reference.md`
2. **SKILL.md 格式**：
   - 必须包含 YAML frontmatter（name 和 description）
   - description 需说明触发条件和关键特征
   - 正文包含说明、使用时机、执行步骤、工具调用等章节
   - **必须明确告知：工具调用出错时优先读取 `references/api_reference.md`**
3. **api_reference.md 格式**：
   - 使用 Markdown 代码块展示接口调用示例
   - 入参和出参使用 JSON 格式说明
   - 添加注释解释关键字段含义
