# 验证模块：答案质量检查

## 目标
确保每个工具返回的答案满足质量门槛，不将不合格答案传递给用户或后续步骤。

## 验证流程

**步骤一：提取关键信息**
从工具返回结果中提取：数值、条款编号、时间戳、信息来源等关键要素

**步骤二：多维度检查**
按照以下四个维度逐项验证：
- **完整性**：是否完整回答子问题的所有要素
- **准确性**：数据是否准确，无 hallucination
- **相关性**：是否直接回答问题，无无关信息
- **一致性**：多工具结果是否逻辑一致

**步骤三：结果判定**
- 全部检查通过 → 标记为 VERIFIED，进入下一步
- 任一检查失败 → 标记为 FAILED，记录失败原因，进入 error.md 处理流程

## 验证维度

### 维度 1：完整性检查（Completeness）
**标准**：答案是否完整回答了子问题的所有要素？

检查清单：
- [ ] 是否回答了问题中的每个关键词？
- [ ] 是否遗漏了问题中的限定条件？（如时间、地域、人群）
- [ ] 对于数值类问题，是否提供了具体数字+单位？

**示例**：
- 问题："30岁男性，50万保额，年缴保费多少？"
- 不合格答案："大概几千块钱"（缺少精确数值）
- 合格答案："年缴保费为 8,500 元"（具体数值+单位）

### 维度 2：准确性检查（Accuracy）
**标准**：答案是否准确，无 hallucination？

检查清单：
- [ ] 保险条款引用是否标注了具体条款编号？
- [ ] 数值是否来自内部工具返回的原始数据（未经过 LLM 改写）？
- [ ] 外部搜索信息是否标注了来源和时间？

**特殊规则**：保险数值类答案必须与工具返回的原值一致，禁止 LLM 进行：
- 单位换算（除非用户明确要求）
- 四舍五入（除非工具已提供近似值）
- 语言改写（保持原始数据精度）

### 维度 3：相关性检查（Relevance）
**标准**：答案是否直接回答问题，无无关信息？

检查清单：
- [ ] 是否包含与问题无关的营销内容？
- [ ] 是否回答了用户没问但"可能有用"的信息？（如果包含，必须明确标注为"补充信息"）
- [ ] 外部搜索结果是否来自权威来源？（保险问题优先官方渠道）

### 维度 4：一致性检查（Consistency）
**标准**：多工具结果是否逻辑一致？

检查清单：
- [ ] 内部保险工具返回的产品名与外部搜索的产品名是否指同一产品？
- [ ] 不同来源的数值是否冲突？（如内部费率与外部报道不一致）
- [ ] 时间信息是否一致？（如外部新闻日期与内部条款生效日期）

## 验证失败类型与处理

| 失败类型 | 示例 | 处理方式 |
|---------|------|---------|
| 信息不完整 | 缺少费率表中的某个年龄段 | 重新查询或明确告知用户数据缺失 |
| 数值不一致 | 内部工具显示 8000，外部显示 7500 | 优先信任内部工具，标注差异 |
| 来源不可靠 | 外部搜索返回论坛匿名帖 | 过滤掉该信息，仅使用官方来源 |
| 答非所问 | 问A产品，返回B产品信息 | 重新执行查询，检查输入参数 |
| 过时信息 | 外部搜索返回 2023 年已废止政策 | 标注时间，提醒用户核实最新政策 |

## 验证输出格式

```yaml
validation_result:
  sub_question_id: 1
  status: VERIFIED | FAILED
  checks:
    completeness: PASS | FAIL (原因)
    accuracy: PASS | FAIL (原因)
    relevance: PASS | FAIL (原因)
    consistency: PASS | FAIL (原因)
  issues: ["问题1", "问题2"]  # 如果 FAILED
  action: CONTINUE | RETRY | FALLBACK | ESCALATE
```

## Gotchas（验证阶段陷阱）

1. **过度信任外部搜索**：外部搜索结果可能过时或与内部条款冲突，必须做一致性检查
   - 陷阱：用户问"最新重疾险定义"，外部搜索返回 2021 年旧定义
   - 防护：检查外部信息时间戳，与内部条款版本对比

2. **数值精度丢失**：LLM 倾向于将"8500.00"改写为"8500元"，可能丢失精度
   - 陷阱：保费计算需要精确到分，改写后导致用户误解
   - 防护：数值类答案直接引用工具返回原值，不做语言化改写

3. **隐性假设**：工具返回了信息，但没验证是否回答用户具体问题
   - 陷阱：用户问"30岁保费"，工具返回"20-40岁费率表"，看似相关但未精准回答
   - 防护：必须提取具体数值，不能仅返回表格让用户自己找

4. **多源冲突忽略**：内部工具与外部搜索矛盾时，未做标记直接选用一方
   - 陷阱：外部说"已停售"，内部显示"在售"，直接采信外部导致错误
   - 防护：冲突必须上报，优先采信内部工具但需标注矛盾点
