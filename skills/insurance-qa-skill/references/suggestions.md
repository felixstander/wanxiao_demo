# 关联问题推荐模块

## 目标
基于用户当前问题和答案内容，生成 2-3 个不同方向的关联问题，引导用户进行下一步咨询，实现从通用信息到保险产品的自然过渡。

## 推荐策略

### 策略 1：热点事件 → 保险产品
**触发条件**：用户询问实时热点、新闻事件

**关联方向**：从事件风险点延伸至相关保险保障

**示例映射**：
| 热点事件关键词 | 推荐问题 |
|---------------|---------|
| 心肌炎、猝死、心脏病 | "是否需要了解心脏疾病相关的重疾险？" |
| 交通事故、车祸 | "您是否配置了充足的意外险保障？" |
| 癌症、肿瘤 | "想了解癌症专项医疗保险吗？" |
| 教育、留学、学费 | "需要为子女教育储备教育金保险吗？" |
| 养老、退休金 | "想了解商业养老保险补充社保吗？" |
| 房价、房贷 | "是否需要房贷保障型定期寿险？" |

### 策略 2：保险产品 → 产品咨询
**触发条件**：用户已获取某产品信息

**关联方向**：深入了解产品细节或关联产品

**示例映射**：
| 当前产品 | 推荐问题 |
|---------|---------|
| 重疾险 | "想了解该重疾险的轻症赔付比例吗？" |
| 医疗险 | "是否需要搭配重疾险形成完整保障？" |
| 意外险 | "想了解含猝死责任的意外险吗？" |
| 寿险 | "需要了解可附加豁免保费的版本吗？" |
| 年金险 | "想了解保底利率和分红演示吗？" |

### 策略 3：保险产品 → 产品对比
**触发条件**：用户询问单一产品

**关联方向**：提供同类产品的对比选择

**示例映射**：
| 当前产品 | 推荐问题 |
|---------|---------|
| A公司重疾险 | "想对比B公司同类型重疾险的费率差异吗？" |
| 定期寿险 | "想了解终身寿险和定期寿险的区别吗？" |
| 百万医疗险 | "想对比0免赔额的医疗险产品吗？" |

### 策略 4：通用咨询 → 个性化方案
**触发条件**：用户进行开放式咨询

**关联方向**：引导至个性化需求分析

**推荐问题模板**：
- "根据您的情况，需要为您定制家庭保障方案吗？"
- "想了解如何用最少的预算配置全面保障吗？"
- "需要评估您现有保单是否存在保障缺口吗？"

## 推荐原则

### 原则 1：关联性但不重复
- 推荐问题必须与当前话题有关联，但不能是同一问题的重复
- 应该提供新的信息价值或方向

### 原则 2：自然过渡
- 从用户已知信息自然延伸到未知领域
- 避免突兀的产品推销感

### 原则 3：控制数量
- 每次推荐 2-3 个问题
- 问题之间应该覆盖不同方向（如一个产品深化、一个横向扩展、一个个性化）

### 原则 4：尊重用户意图
- 如果用户明确表示"只是了解一下"，减少推荐强度
- 如果用户表现出强烈兴趣，可以增加产品化推荐

## 生成规则

### 规则 1：基于内容分析
- 提取答案中的关键词（疾病、事件、产品名称等）
- 根据关键词匹配对应的推荐策略

### 规则 2：多策略组合
- 优先使用最匹配的策略
- 可以同时混合使用两种策略（如热点+个性化）

### 规则 3：去重过滤
- 不推荐用户已经问过的问题
- 不推荐与当前问题 80% 相似的问题

### 规则 4：优先级排序
1. 与当前话题直接相关的产品推荐
2. 横向扩展的关联产品
3. 个性化方案建议

## 输出格式

在最终答案后追加推荐问题模块：

```markdown
---

**您可能还想了解：**

1. [推荐问题1]？
2. [推荐问题2]？
3. [推荐问题3]？
```

## 扩展指南

### 添加新策略
如需增加新的推荐策略（如"健康问答 → 体检预约"）：

1. 在"推荐策略"章节添加新章节：
   ```markdown
   ### 策略 N：健康问答 → 体检服务
   **触发条件**：用户询问健康知识
   **关联方向**：从健康风险延伸至预防性检查
   ```

2. 添加示例映射表

3. 在 SKILL.md 执行流程中注明何时使用该策略

### 修改现有策略
直接编辑对应策略章节的示例映射表即可，无需修改其他文件。

## Gotchas

1. **过度推荐**：频繁推荐会让用户感到被打扰，仅在完整回答后推荐一次
2. **策略冲突**：多个策略同时触发时，选择匹配度最高的，或组合不同方向的问题
3. **无关推荐**：确保推荐问题与当前内容有逻辑关联，避免随机推荐
4. **忽视用户反馈**：如果用户多次忽略推荐问题，后续应减少推荐强度
