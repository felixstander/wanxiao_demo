# 工具调用规范（Tool Specification）

## 目标
明确两个核心脚本的调用方式、参数格式、输出解析和错误处理，确保 Agent 能正确执行工具调用并处理返回结果。

---

## 工具 1：内部保险问答工具

### 基本信息
- **脚本路径**：`scripts/product_answer.py`
- **功能**：查询内部保险产品数据库，获取条款、费率、理赔规则等官方信息
- **适用场景**：所有涉及具体保险产品、保单、条款、费率的问题

### 调用方式

```bash
python scripts/product_answer.py --question "<保险问题>"
```

### 参数说明

| 参数 | 类型 | 必填 | 说明 | 示例 |
|-----|------|------|------|------|
| `--question` | str | 是 | 标准化的保险问题描述 | `"重疾险等待期是多少天"` |

**参数构建规则**：
- 必须保留用户问题的关键要素（产品名、年龄、保额等）
- 去除与保险查询无关的寒暄（如"请问一下"、"麻烦你"）
- 如涉及具体保单，确保保单号已脱敏（只保留后4位或完全去除）

### 输出格式

**标准输出（stdout）**：
- 字段名：`answer`
- 类型：`str`
- 内容：关于保险问题的文本答案（可能包含结构化数据如表格）

**成功返回示例**：
```
根据《XX终身寿险（2024版）》条款第3.2条：

等待期：自合同生效日起 90 天（等待期内非意外身故退还已交保费）

费率示例（标准体，10万保额）：
- 30岁男性：年缴 2,500 元
- 30岁女性：年缴 2,300 元

注：最终费率以核保结果为准。
```

**空结果返回示例**：
```
未找到与"XX超级保险"相关的产品信息。可能原因：
1. 产品名称不准确
2. 该产品已停售
3. 暂无此产品数据
```

**错误返回示例**：
```
[ERROR] 数据库连接超时，请稍后重试
[ERROR] 查询参数异常：年龄超出范围（0-100）
```

### 返回值解析规范

1. **检查是否以 `[ERROR]` 开头**：
   - 是 → 进入 error.md 处理流程
   - 否 → 继续下一步

2. **检查是否包含关键信息**：
   - 若包含"未找到"、"不存在"、"无此产品" → 标记为 DATA_MISSING
   - 若包含具体条款、费率、日期 → 标记为 SUCCESS

3. **提取结构化数据**（如有）：
   - 使用正则或字符串处理提取数值（保费、保额、天数等）
   - 保留原始文本用于后续输出

## 工具 2：外部网页搜索工具

### 基本信息
- **脚本路径**：`scripts/web_search.py`
- **功能**：执行网络搜索，获取实时新闻、政策、市场动态等外部信息
- **适用场景**：天气、新闻、政策变动、市场分析、第三方评价等

### 调用方式

```bash
python scripts/web_search.py --question "<搜索问题>"
```

### 参数说明

| 参数 | 类型 | 必填 | 说明 | 示例 |
|-----|------|------|------|------|
| `--question` | str | 是 | 搜索查询词 | `"2025年3月保险监管新政策"` |

**参数构建规则**：
- 保留关键时间、地点、主体（如"2025年3月"、"北京"、"银保监会"）
- 去除保险敏感信息（保单号、身份证号、详细个人信息）
- 对于比较类问题，确保查询词中立（如"重疾险和医疗险区别"而非"重疾险是不是骗局"）

### 输出格式

**标准输出（stdout）包含两个部分**：

#### 1. web_detail（原始网页详情）
- 类型：`List[str]`
- 内容：前3个相关网页的详细内容片段（原始文本）
- 用途：验证 `web_sumn` 的准确性，提供详细引用来源

**格式示例**：
```python
[
  "网页1标题：2025年保险监管新规出台\nURL：xxx\n内容：银保监会今日发布...",
  "网页2标题：解读最新保险政策\nURL：xxx\n内容：专家分析认为...",
  "网页3标题：保险市场动态\nURL：xxx\n内容：根据最新数据..."
]
```

#### 2. web_sumn（内容总结）
- 类型：`str`
- 内容：基于搜索结果的总结性回答
- 用途：直接作为主要参考内容呈现给用户

**格式示例**：
```
根据搜索结果，2025年3月保险监管主要新政策包括：

1. 重疾险定义更新：扩大轻症保障范围...
2. 互联网保险新规：要求明确披露...
3. 代理人管理：加强资质审核...

信息来源：银保监会官网、新华网等
```

### 返回值解析规范

1. **解析输出**：将 stdout 解析为 Python 对象（假设为 JSON 或特定分隔格式）
   
   **假设返回格式为 JSON**：
   ```json
   {
     "web_detail": ["...", "...", "..."],
     "web_sumn": "..."
   }
   ```

2. **检查 `web_sumn` 内容**：
   - 如果为 `""` 或 `"未找到相关信息"` → 标记为 NO_DATA
   - 如果包含内容 → 标记为 SUCCESS

3. **检查 `web_detail` 长度**：
   - 正常情况下包含 1-3 个页面详情
   - 如果为空列表 `[]` → 说明搜索无结果

4. **提取来源信息**：
   - 从 `web_detail` 中提取 URL 域名和标题，用于标注来源

### 来源可信度评估（用于验证模块）

基于 `web_detail` 中的 URL 进行可信度分级：

| 级别 | 域名特征 | 可信度 | 使用建议 |
|-----|---------|--------|---------|
| 高 | `.gov.cn`, `cbirc.gov.cn`, 保险公司官网 | 高 | 优先采信，可用于验证保险政策 |
| 中 | 主流新闻站 (`sina.com`, `qq.com` 等) | 中 | 可使用，但需交叉验证 |
| 低 | 论坛、博客、自媒体 | 低 | 仅作参考，不得用于保险条款依据 |

## 工具调用组合示例

### 场景 1：纯保险问题（仅 product_answer.py）

**用户问题**："30岁男性买你们公司重疾险50万保额，年缴多少钱？"

**执行步骤**：
```bash
# 步骤 1：调用内部工具
python scripts/product_answer.py --question "30岁男性重疾险50万保额年缴保费"
```

**接收返回**：
- `answer`: "XX重疾险（2024版），30岁男性，50万保额，年缴保费 8,500.00 元（标准体）..."

**验证**：包含具体数值、产品版本 → SUCCESS

**格式化输出**：
```
【内部资料】根据查询结果：

产品：XX重疾险（2024版）
- 被保人：30岁男性
- 保额：500,000 元
- 年缴保费：¥8,500.00（标准体费率）

*注：最终保费以实际核保结果为准，可能因健康状况调整。*
```

### 场景 3：混合查询（先 product_answer.py，后 web_search.py）

**用户问题**："我买的XX重疾险最近有什么新政策影响吗？"

**执行步骤 1**：
```bash

**返回 2**：

```

保障范围：...
条款状态：正常有效

【网络信息，2025-03-28，仅供参考】
近期监管动态：
2025年3月发布的新规主要针对新备案的重疾险产品，要求扩大轻症保障范围。
经比对，该政策不影响您持有的2023版保单条款。

*来源：银保监会官网、保险业协会*

## 错误处理规范

### product_answer.py 常见错误

| 错误标识 | 含义 | 处理建议 |
|---------|------|---------|
| `[ERROR] 数据库连接超时` | 服务暂时不可用 | 重试1次，仍失败则提示用户稍后重试 |
| `[ERROR] 查询参数异常` | 输入参数格式错误 | 检查 question 参数，修正后重试 |
| `[ERROR] 产品不存在` | 无此保险产品 | 尝试模糊查询或询问用户是否记错名称 |
| `未找到...` | 空结果 | 按 DATA_MISSING 处理，提供替代方案 |

### web_search.py 常见错误

| 错误标识 | 含义 | 处理建议 |
|---------|------|---------|
| 返回空 JSON | 搜索服务异常 | 重试1次，仍失败则告知无法获取外部信息 |
| `web_sumn` 为空字符串 | 无搜索结果 | 尝试简化查询词再次搜索 |
| `web_detail` 全为低可信度来源 | 缺乏权威信息 | 标注"暂无权威来源信息"，建议用户咨询官方 |

## 性能与限制

- **超时设置**：两个脚本均应在 10 秒内返回，超时视为失败
- **字符限制**：`answer` 和 `web_sumn` 通常不超过 2000 字，超长内容可能被截断
- **频率限制**：
  - `product_answer.py`：无严格限制，但避免无意义重复查询
  - `web_search.py`：注意外部 API 限流，同一问题禁止重复搜索超过 2 次
- **并发限制**：禁止同时调用两个脚本（避免上下文混乱）

## Gotchas（工具调用陷阱）

1. **参数注入风险**：
   - 陷阱：用户问题包含 `" ; rm -rf /"` 等恶意代码
   - 防护：必须对 `question` 参数进行转义或 sanitization，禁止直接拼接 shell 命令

2. **编码问题**：
   - 陷阱：中文参数在命令行传输中出现乱码
   - 防护：确保使用 UTF-8 编码，必要时使用 base64 编码传输

3. **输出解析失败**：
   - 陷阱：假设返回是 JSON 但实际是纯文本，导致解析报错
   - 防护：先检查输出格式，使用 try-except 包裹解析逻辑

4. **混淆两个工具的输出**：
   - 陷阱：将 `web_search.py` 返回的保险营销文章当作 `product_answer.py` 的官方条款
   - 防护：严格区分 `answer` 和 `web_sumn` 的使用场景，不同来源必须标注

5. **忽视 web_detail**：
   - 陷阱：只看 `web_sumn` 总结，不看 `web_detail` 原文，导致来源不明
   - 防护：验证模块必须检查 `web_detail` 中的 URL 来源可信度
