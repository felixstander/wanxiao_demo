# 报错/冲突处理模块

## 目标
优雅处理工具调用失败、返回异常、信息冲突等情况，确保不向用户传递错误技术信息，同时提供可行的 fallback 方案。

## 错误分类与处理策略

### 类别 1：工具调用失败（Technical Errors）

**场景**：
- 内部保险工具超时/无响应
- 外部搜索 API 限流/错误
- 网络连接问题

**处理流程**：

1. **重试机制**：
   - `product_content_search`/`product_attribute_detail`/`product_tool_interface`：最多重试 2 次，间隔 1 秒
   - `web_search`：最多重试 1 次（避免触发限流）

2. **重试失败 → 降级策略**：
   - **内部工具失败**：
     - 尝试切换同类型工具（如`product_attribute_detail`失败时尝试`product_content_search`）
     - 仍失败：明确告知用户"系统暂时无法查询保险资料，建议稍后重试或联系客服"
   - **外部搜索失败**：
     - 如果问题是保险相关的，尝试用内部工具回答基础部分
     - 如果是纯外部信息，告知用户"暂时无法获取最新信息"

3. **禁止行为**：
   - 禁止用`web_search`替代内部工具回答保险问题
   - 禁止编造数据"假装"工具返回成功
   - 禁止向用户暴露技术错误信息（如SQL错误、堆栈跟踪等）

**输出模板**：
```
【系统提示】
暂时无法查询到 [具体信息类型]，可能原因：[简要说明]
建议：[具体行动建议，如"稍后重试"/"拨打客服热线 95511"]
```

### 类别 2：数据缺失（Data Missing）

**场景**：
- 工具返回空结果（产品不存在/已停售）
- 查询条件过于具体，无匹配数据
- 产品编码/名称错误

**处理流程**：

1. **确认查询参数**：
   - 检查产品名称是否拼写错误（如"重级险" vs "重疾险"）
   - 检查产品编码格式是否正确
   - 检查是否包含不可能的限定条件（如"100岁购买少儿险"）

2. **扩大查询范围**：
   - 使用`product_content_search`的模糊搜索（`--productName`支持模糊匹配）
   - 列出该类别所有产品供用户选择
   - 尝试不带`effectiveState`筛选，查看是否存在但已停售

3. **仍无结果 → 明确告知**：
   ```
   未找到名为'XX'的产品，是否指以下产品：
   - 相似产品A
   - 相似产品B
   
   如您不确定产品名称，可以告诉我您想了解的保险类型（如重疾险、医疗险等）。
   ```

### 类别 3：信息冲突（Conflicts）

**场景**：
- 不同内部工具返回的产品信息不一致
- 内部资料与外部搜索结果矛盾
- 同一产品的不同版本信息冲突
- 不同外部来源信息矛盾

**处理优先级**（优先级从高到低）：
1. **`product_tool_interface`**（责任详情）> `product_attribute_detail` > `product_content_search`
2. **内部工具** > **外部搜索**（保险条款、费率、状态等核心信息）
3. **官方监管/公司官网** > **主流新闻** > **论坛/自媒体**（外部信息）
4. **最新生效版本** > **旧版本**（同一产品多版本）

**冲突处理流程**：
```
发现冲突
    ↓
标记冲突点（记录双方信息+来源+时间）
    ↓
按优先级判断可信度
    ↓
高优先级可信？
    ├─ 是 → 采用高优先级，但在答案中标注冲突
    └─ 否 → 标记为 UNRESOLVED，建议人工核实
```

**输出模板（冲突情况）**：
```
【信息说明】
关于 [具体问题]，不同来源信息存在差异：

✓ 内部资料（[工具名称]，[日期]）：[高优先级信息]
⚠ 其他来源（[来源]，[日期]）：[低优先级信息]

建议以官方最新公告为准，或联系客服核实：95511
```

### 类别 4：工具返回不完整

**场景**：
- `product_attribute_detail`返回的属性信息缺失某些字段
- `product_tool_interface`查询的字段不存在
- `web_search`返回的`web_sumn`为空但`web_detail`有内容

**处理流程**：

1. **检查缺失字段重要性**：
   - 核心字段缺失（如产品名称、保障责任）：尝试其他工具补充
   - 非核心字段缺失（如销售渠道）：继续执行，在输出中标注"信息待补充"

2. **尝试补充查询**：
   - 使用其他工具查询缺失信息
   - 调整参数后重新查询

3. **输出模板**：
   ```
   已获取以下信息：
   - 产品名称：XX重疾险
   - 投保年龄：18-55周岁
   
   以下信息暂未获取：[缺失字段列表]
   如需了解详细信息，建议联系客服：95511
   ```

### 类别 5：逻辑不一致（Logical Inconsistency）

**场景**：
- 子问题1说"30岁可投保"，子问题2说"该产品仅限18-25岁"
- 同一产品的投保规则前后矛盾

**处理流程**：
1. **标记逻辑矛盾**
2. **回溯检查**：是否调用了不同版本的产品？是否参数设置错误？
3. **无法解释的矛盾**：不合并答案，分别呈现并标注矛盾点
4. **建议人工复核**

## 错误恢复策略矩阵

| 错误场景 | 自动重试 | 切换工具 | 人工提示 | 终止流程 |
|---------|---------|---------|---------|---------|
| 工具超时 | ✓ 2次 | ✓（同类型工具） | ✓ | 最终手段 |
| 空结果 | ✗ | ✓（扩大查询） | ✓ | ✗ |
| 内外部冲突 | ✗ | ✗ | ✓ 必须标注 | ✗ |
| 逻辑矛盾 | ✗ | ✗ | ✓ 必须标注 | 严重时可终止 |
| 敏感信息泄露风险 | ✗ | ✗ | ✗ | ✓ 立即终止 |

## 常见错误及处理

### product_content_search 常见错误

| 错误标识 | 含义 | 处理建议 |
|---------|------|---------|
| `[ERROR] 参数格式错误` | contentType等参数值不在枚举范围内 | 检查参数值是否在允许的枚举列表中 |
| `[ERROR] 未找到匹配产品` | 无符合条件的产品 | 尝试移除部分筛选条件或扩大topK |
| `返回空列表` | 查询条件过于严格 | 逐步移除筛选条件重试 |

### product_attribute_detail 常见错误

| 错误标识 | 含义 | 处理建议 |
|---------|------|---------|
| `[ERROR] productName和productCode至少填一个` | 缺少必填参数 | 补充产品名称或编码 |
| `[ERROR] 产品编码不存在` | productCode错误 | 先通过productName查询正确编码 |
| `[ERROR] 属性类型不存在` | attributeType值错误 | 检查是否在允许的枚举列表中 |

### product_tool_interface 常见错误

| 错误标识 | 含义 | 处理建议 |
|---------|------|---------|
| `[ERROR] mktProdCode不能为空` | 缺少必填参数 | 补充产品编码 |
| `[ERROR] 产品不存在` | 编码错误 | 通过其他工具查询正确编码 |
| `[ERROR] 字段不存在` | requestFields包含无效字段 | 检查字段名称是否正确 |

### web_search 常见错误

| 错误标识 | 含义 | 处理建议 |
|---------|------|---------|
| `[ERROR] 搜索服务异常` | 外部服务问题 | 重试1次，仍失败则告知无法获取 |
| `web_sumn为空` | 无搜索结果 | 尝试简化查询词 |
| `全为低可信度来源` | 缺乏权威信息 | 标注"暂无权威来源" |

## Gotchas（错误处理陷阱）

1. **静默失败**
   - 陷阱：工具返回空结果或错误，但系统"假装"成功继续执行
   - 示例：`product_attribute_detail`返回空，系统用假设的数据继续
   - 防护：必须检查工具返回状态，空结果或错误状态一律进入错误处理

2. **冲突掩盖**
   - 陷阱：发现冲突但选择"简单处理"只呈现一方
   - 示例：内部显示在售，外部显示停售，只告诉用户"在售"
   - 防护：任何冲突必须在答案中显性标注，不能隐瞒

3. **过度重试**
   - 陷阱：无限重试导致响应时间过长或触发限流
   - 示例：外部搜索失败后不断重试，导致IP被封
   - 防护：严格限制重试次数，失败后立即降级

4. **错误信息传递**
   - 陷阱：将工具原始错误信息直接暴露给用户
   - 示例："SQL Error: Table not found"直接显示给用户
   - 防护：所有错误必须转化为用户友好的提示

5. **工具切换不当**
   - 陷阱：`product_tool_interface`失败后切换到`web_search`查询保险信息
   - 示例：查询责任详情失败，改用外部搜索获取
   - 防护：工具切换只能在同类工具间进行，禁止用外部搜索替代内部工具
