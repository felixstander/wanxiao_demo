# 报错/冲突处理模块

## 目标
优雅处理工具调用失败、返回异常、信息冲突等情况，确保系统不向用户传递错误信息，同时提供可行的 fallback 方案。

## 错误分类与处理策略

### 类别 1：工具调用失败（Technical Errors）

**场景**：
- 内部保险工具超时/无响应
- 外部搜索 API 限流/错误
- 网络连接问题

**处理流程**：
1. **重试机制**：
   - 内部工具：最多重试 2 次，间隔 1 秒
   - 外部搜索：最多重试 1 次（避免触发限流）
   
2. **重试失败 → 降级策略**：
   - 内部工具失败：明确告知用户"系统暂时无法查询保险资料，建议稍后重试或联系客服"
   - 外部搜索失败：如果问题是保险相关的，尝试用内部工具回答基础部分；如果是纯外部信息，告知用户"暂时无法获取最新信息"

3. **禁止行为**：
   - 禁止用外部搜索替代内部工具回答保险问题
   - 禁止编造数据"假装"工具返回成功

**输出模板**：
```
【系统提示】
暂时无法查询到 [具体信息类型]，可能原因：[简要说明]
建议：[具体行动建议，如"稍后重试"/"拨打客服热线 400-xxx"]
```

### 类别 2：数据缺失（Data Missing）

**场景**：
- 内部工具返回空结果（产品不存在/已停售）
- 查询条件过于具体，无匹配数据

**处理流程**：
1. **确认查询参数**：
   - 检查是否拼写错误（如"重级险" vs "重疾险"）
   - 检查是否包含不可能的限定条件（如"100岁购买少儿险"）

2. **扩大查询范围**：
   - 如果查询具体产品失败，尝试列出该类别所有产品
   - 如果带条件查询失败，尝试无条件下查询，再人工筛选

3. **仍无结果 → 明确告知**：
   - "未找到名为'XX'的产品，是否指以下产品：[相似产品列表]"
   - "您描述的保障需求暂无完全匹配产品，建议咨询顾问定制方案"

### 类别 3：信息冲突（Conflicts）

**场景**：
- 内部工具显示产品A在售，外部搜索显示已停售
- 内部费率与外部报道不一致
- 不同外部来源信息矛盾

**处理优先级**：
1. **保险条款、费率、状态**：优先采信**内部工具**
2. **市场新闻、政策解读**：优先采信**官方监管/公司官网**
3. **用户评价、非官方解读**：仅作参考，必须标注来源

**冲突处理流程**：
发现冲突
   ↓
标记冲突点（记录双方信息+来源+时间）
   ↓
优先级判断
   ↓
高优先级可信？→ 是：采用高优先级，但必须在答案中标注冲突
   ↓
否 → 标记为 UNRESOLVED，不回答该子问题，建议人工核实

**输出模板（冲突情况）**：
```
【信息说明】
关于 [具体问题]，不同来源信息存在差异：

✓ 内部资料（[日期]）：[内部工具返回的信息]
⚠ 网络信息（[来源]，[日期]）：[外部搜索返回的信息]

建议以官方最新公告为准，或联系客服核实：[客服方式]
```

### 类别 4：逻辑不一致（Logical Inconsistency）

**场景**：
- 子问题1说"30岁可投保"，子问题2说"该产品仅限18-25岁"
- 保费计算结果与保额不成比例（如50万保额比30万便宜）

**处理流程**：
1. **标记逻辑矛盾**
2. **回溯检查**：是否误解了用户问题？是否调用了错误的产品代码？
3. **无法解释的矛盾**：不合并答案，分别呈现并标注矛盾点
4. **建议人工复核**

## 错误恢复策略矩阵

| 错误场景 | 自动重试 | 切换工具 | 人工提示 | 终止流程 |
|---------|---------|---------|---------|---------|
| 工具超时 | ✓ 2次 | ✗ | ✓ | 最终手段 |
| 空结果 | ✗ | ✓（扩大查询） | ✓ | ✗ |
| 内外部冲突 | ✗ | ✗ | ✓ 必须标注 | ✗ |
| 逻辑矛盾 | ✗ | ✗ | ✓ 必须标注 | 严重时可终止 |
| 敏感信息泄露风险 | ✗ | ✗ | ✗ | ✓ 立即终止 |

## Gotchas（错误处理陷阱）

1. **静默失败**：工具返回空结果或错误，但系统"假装"成功继续执行
   - 陷阱：内部工具报错，系统用外部搜索替代，导致错误信息
   - 防护：必须检查工具返回状态码，非成功状态一律进入错误处理

2. **冲突掩盖**：发现冲突但选择"简单处理"只呈现一方
   - 陷阱：内部显示在售，外部显示停售，只告诉用户"在售"，导致用户无法购买
   - 防护：任何冲突必须在答案中显性标注，不能隐瞒

3. **过度重试**：无限重试导致响应时间过长或触发限流
   - 陷阱：外部搜索失败后不断重试，导致 IP 被封
   - 防护：严格限制重试次数，失败后立即降级

4. **错误信息传递**：将工具原始错误信息直接暴露给用户
   - 陷阱："SQL Error: Table not found" 直接显示给用户
   - 防护：所有错误必须转化为用户友好的提示，技术细节记录日志
